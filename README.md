# Route Analyzer

Find every aircraft type that has ever flown between any two airports — powered by Claude AI.

Enter an origin and destination (IATA or ICAO codes) and get back the aircraft types historically and currently operated on that route, grouped by category and era, along with the airlines that fly them, the great-circle distance, and notes about the route.

## How it works

The frontend collects two airport codes and POSTs them to the backend. The backend prompts Claude (`claude-sonnet-4-6`) to act as an aviation data expert and return a structured JSON payload describing the route and its aircraft. The frontend renders that payload as a set of aircraft cards.

```
React + Vite client  ──POST /api/aircraft──▶  Express server  ──▶  Anthropic API (Claude)
```

## Prerequisites

- Node.js 20.x
- An Anthropic API key ([console.anthropic.com](https://console.anthropic.com))

## Local development

The client and server run as two processes in development.

### 1. Backend

```bash
cd server
npm install
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY=sk-ant-...
npm run dev
```

The server runs on http://localhost:3001 (`npm run dev` uses `node --watch` for auto-reload).

### 2. Frontend (in a new terminal)

```bash
cd client
npm install
npm run dev
```

The app runs on http://localhost:5173. The Vite dev server proxies `/api` requests to the backend, so no extra CORS configuration is needed during development.

## Environment variables

Set these in `server/.env` (see `server/.env.example`):

| Variable            | Description                        | Default |
| ------------------- | ---------------------------------- | ------- |
| `ANTHROPIC_API_KEY` | Your Anthropic API key (required)  | —       |
| `PORT`              | Port the Express server listens on | `3001`  |

## API

### `POST /api/aircraft`

**Request body**

```json
{ "origin": "LAX", "destination": "JFK" }
```

**Response**

```json
{
  "originName": "Los Angeles International Airport, Los Angeles",
  "destName": "John F. Kennedy International Airport, New York",
  "routeExists": true,
  "distance_km": 3974,
  "aircraft": [
    {
      "type": "Boeing 737-800",
      "iata": "738",
      "icao": "B738",
      "category": "Narrowbody",
      "airlines": ["Delta Air Lines", "American Airlines"],
      "era": "current",
      "notes": ""
    }
  ],
  "routeNotes": "Major transcontinental route with frequent daily service."
}
```

`category` is one of `Narrowbody`, `Widebody`, `Regional Jet`, `Turboprop`, or `Business Jet`. `era` is one of `current`, `historical`, or `both`. If the route is implausible or the codes are invalid, `routeExists` is `false` and `aircraft` is empty.

## Production build & deployment

In production a single Node process serves both the API and the static client build. The Express server serves `client/dist` and falls back to `index.html` for client-side routing.

```bash
npm run build   # installs client deps and builds client/dist
npm start       # node server/index.js
```

### Heroku

The repo is configured for Heroku out of the box:

- `Procfile` declares the web process (`node server/index.js`).
- `heroku-postbuild` installs server and client dependencies and builds the client.
- `engines.node` pins Node 20.x.

Set the `ANTHROPIC_API_KEY` config var on the app, then deploy:

```bash
heroku config:set ANTHROPIC_API_KEY=sk-ant-...
git push heroku main
```

## Tech stack

- **Frontend:** React 18, Vite 5
- **Backend:** Express 4, `@anthropic-ai/sdk`
- **AI:** Claude (`claude-sonnet-4-6`)

## Note

Aircraft and route data is generated by Claude and reflects the model's knowledge — treat results as informative rather than authoritative.
